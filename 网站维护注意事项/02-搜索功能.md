## 一、前端搜索表单（functions.php 第 34-36 行）

```html
<form class="top-search" action="/" method="get">
    <input type="text" name="s" placeholder="搜索档案、事件、人物..."
           required
           pattern=".*[a-zA-Z0-9\u4e00-\u9fff].*"
           title="请至少输入一个字母、数字或中文">
    <button type="submit">搜索</button>
</form>
```

### 校验规则

| 层级 | 机制 | 拦截内容 |
|------|------|----------|
| 前端 `required` | 浏览器原生 | 空输入 |
| 前端 `pattern` | 正则 `.*[a-zA-Z0-9\u4e00-\u9fff].*` | 纯空格、纯符号（如 `@#$%`） |
| 后端 `template_redirect` | PHP `preg_match` | 绕过前端的非法请求 |

## 二、后端拦截逻辑（functions.php 第 213-226 行）

```php
add_action('template_redirect', function () {
    if (is_search() && isset($_GET['s'])) {
        $s = trim($_GET['s']);
        // 1. 空搜索 → 回首页
        if ($s === '') {
            wp_redirect(home_url('/')); exit;
        }
        // 2. 彩蛋：搜索 "CRRG-917" → 跳转管理面板
        if (strtolower($s) === 'crrg-917') {
            wp_redirect(home_url('/admin/')); exit;
        }
        // 3. 纯符号 → 回首页
        if (!preg_match('/[a-zA-Z0-9\x{4e00}-\x{9fff}]/u', $s)) {
            wp_redirect(home_url('/')); exit;
        }
    }
    // 4. 404 /register → 跳转注册页
    if (is_404() && $_SERVER['REQUEST_URI'] === '/register') {
        wp_redirect('/wp-login.php?action=register'); exit;
    }
});
```

### 拦截优先级

1. 空字符串 → 重定向首页
2. `crrg-917`（大小写不敏感）→ 重定向 `/admin/`（管理面板彩蛋）
3. 不含任何字母/数字/中文 → 重定向首页
4. `/register` 404 → 跳转 WordPress 注册页

## 三、搜索结果页（search.php）

### 搜索范围

默认同时搜索两种内容类型：
- **`post`**（文章，含归档新闻）
- **`topic`**（论坛话题）

> 不搜索 `page`（页面）

### Tab 切换

搜索结果顶部有三个 tab：
- **全部** — 显示文章 + 论坛话题混合结果
- **文章** — 仅显示 post 类型
- **论坛话题** — 仅显示 topic 类型

点击 tab 通过 URL 参数 `?s=关键词&type=post` 或 `?s=关键词&type=topic` 切换。

## 四、已知问题 & 注意事项

1. **文章数量少**：数据库 `post` 类型只有 2 篇已发布文章（都是归档新闻），所以搜索结果少是正常的，不是搜索功能的问题。
2. **`pattern` 属性限制**：HTML5 `pattern` 仅对表单提交生效，直接访问 URL（如 `/?s=@#$`）只能靠后端拦截。
3. **修改搜索行为**：需要改三处——前端 `pattern`、后端 `preg_match`、`search.php` 模板。
