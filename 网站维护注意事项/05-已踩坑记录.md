# 已踩坑记录 (2026-07-08)

## 1. RSS Feed 浏览器直接打开 → 下载/空白

**现象：**
- Firefox 打开 `/feed/` 下载乱码文件
- XSL 渲染后页面空白

**根因链条：**
1. `Content-Type: application/rss+xml` → Firefox v64+ 直接下载
2. XSL PI 被注入了**两次**：`rss_tag_pre` + `rss2_ns`
   - `rss2_ns` 把 PI 塞进了 `<rss>` 标签属性区（`>` 之前），XML 非法
3. XSL 里用了 `disable-output-escaping="yes"` → Firefox XSLT 引擎报 `8053000e`
4. XSL 文件 MIME 为 `application/octet-stream` → nginx 不识别 `.xsl`

**修复：**
- `feed_content_type` filter → `text/xml`
- 删除 `rss2_ns` hook，仅保留 `rss_tag_pre`
- XSL 改用 `xsl:attribute`，去掉 `disable-output-escaping`
- nginx `mime.types` 添加 `text/xsl xsl xslt`

**教训：**
- WordPress hook 机制要搞清楚插入位置
- `rss2_ns` 在 `<rss ...>` 标签**内部**触发，不是之后
- 先 `xmllint` 验证 XML 再查 XSL 问题

## 2. 央视新闻横栏加载失败

**现象：**
- 页面上方新闻滚动条显示"新闻加载失败"

**根因：**
- `cctv_news_cache.json` 属主为 `root:root`，PHP（www）无法写入
- `file_put_contents()` 报 Warning 输出到 JSON 前面，JSON.parse 失败

**修复：**
- `chown www:www cctv_news_cache.json`
- `@file_put_contents` 加抑制符防止将来再次污染

**教训：**
- 任何 PHP 写的文件都要检查属主
- JSON 接口的前导 Warning/Notice 会直接导致前端崩溃

## 3. 标签系统踩坑

**现象：**
- 文章底部标签不显示
- 提交报告时无法添加标签
- 标签归档 URL（中文）返回 404

**分别的根因：**
1. 标签展示用了 `astra_entry_content_single` hook，Elementor 布局不触发
2. 报告提交页（`/reports/`）表单里没有标签输入框
3. 标签 slug 用了中文 URL 编码（`%e9%87%8d%e7%94%9f...`）

**修复：**
- 改用 `the_content` filter + `ob_start/ob_get_clean` + `return $content . $html`
- `page-reports.php` 三处表单各加标签 input
- 数据库统一 slug 为英文

**教训：**
- 别用 Astra 特有 hook，WordPress 标准 `the_content` 最稳
- `the_content` filter 必须返回修改后的内容，不能直接 echo
- 标签 slug 从一开始就该用英文

## 4. 报告页 ≠ 新闻投稿页

**现象：**
- 改了 `page-news-submit.php` 加标签字段，但用户说看不到

**根因：**
- "提交新报告" → `/reports/`（`page-reports.php`），不是 `/newsroom-submit/`
- 两个完全不同的页面模板

**教训：**
- 先确认用户说的是哪个页面，别猜

## 5. Git 处于 rebase 状态

**现象：**
- `git pull --rebase` 报 "You are not currently on a branch"

**根因：**
- 上次 rebase 没完成（可能网络中断），处于 `rebase in progress`
- `git branch` 显示 `* (no branch, rebasing main)`

**修复：**
```bash
git rebase --abort
git checkout main
git pull --rebase origin main
git push
```
