## RSS Feed 问题排查

```bash
bash /www/wwwroot/crrg_moebius_com_114/siteguardian-skills/scripts/rss-diagnose.sh

# 手动检查
curl -sI https://crrg-moebius.com/feed/ | grep content-type   # 应为 text/xml
curl -s https://crrg-moebius.com/feed/ | head -3              # 应有 xml-stylesheet
```

| 症状 | 原因 | 修复 |
|------|------|------|
| Firefox 下载文件 | Content-Type=`application/rss+xml` | `add_filter('feed_content_type', fn() => 'text/xml')` |
| XSL 渲染空白 | XSL PI 被注入两次（`rss2_ns` 塞进 \<rss\> 标签内） | 只用 `rss_tag_pre`，禁用 `rss2_ns` |
| XSL 报 8053000e | `disable-output-escaping` 触发 Firefox 安全策略 | 改用 `xsl:attribute` |
| XSL 文件返回 octet-stream | nginx 不识别 `.xsl` 扩展名 | mime.types 加 `text/xsl xsl xslt` |

## 紧急排查三步

1. **网站打不开** → `ssh` 进服务器 → `bash scripts/health-check.sh`
2. **功能异常** → 检查 `functions.php` 最近是否被改坏了 → 对比 `.bak` 备份
3. **搜索出问题** → 看 `02-搜索功能.md`
4. **RSS 出问题** → 看上面 RSS 速查表
5. **新闻横栏挂了** → `chown www:www cctv_news_cache.json`

## 改代码流程

```
1. 备份：cp xxx.php xxx.php.bak.$(date +%Y%m%d_%H%M%S)
2. 修改文件
3. 刷新测试
4. 确认无误 → 删旧 bak（保留最近 2 个）
```
